helium — W3C conformance across releases

Share of today's conformance suites passed by each tagged release, measured retroactively.

Data table (raw counts)
Method

Each release's library code is measured unmodified by the current W3C test harness (helium-w3c-tests). Because that harness didn't exist for the older tags and its test glue targets today's API, a small per-tag adapter branch of the harness bridges API differences — it never changes the library under test and never fabricates a passing result; where a release lacks a feature, the harness degrades honestly and those cases fail. The denominator for a suite is the case set the newest release () enumerates with the unmodified harness — i.e. today's full suite. A release's score is its passing cases over that fixed denominator, so cases an old release cannot even enumerate (its parser or schema compiler chokes before running them) count as not-passing.
