helium — W3C conformance across releases

Share of today's conformance suites passed by each tagged release, measured retroactively.

Data table (raw counts)
Method

Each release's library code is measured unmodified by the current W3C test harness (helium-w3c-tests). Because that harness didn't exist for the older tags and its test glue targets today's API, a small per-tag adapter branch of the harness bridges API differences — it never changes the library under test and never fabricates a passing result; where a release lacks a feature, the harness degrades honestly and those cases fail. The denominator for a suite is the set of cases the newest release () actually runs with the unmodified harness: today's suite minus the cases the harness skips as inapplicable — XML 1.0 editions 1–4 where helium implements the 5th, XML 1.1 / Namespaces 1.1 where helium targets 1.0, and the equivalent dependency gates in the other suites. Those cases are not failures and never could be, so charging them to a release would understate every release. A release's score is its passing cases over that fixed denominator, so cases it cannot enumerate (its parser or schema compiler chokes before running them), cases it skips that the reference runs, and cases that hang or exhaust its memory all count as not-passing.
